# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Academic Master's thesis: *Vision-Based Running Technique Classification from In-the-Wild Internet Videos using Markerless Pose Estimation and Deep Learning*. The repo contains the LaTeX source and Python figure-generation scripts.

## Build Commands

**Compile the full thesis (PDF):**
```
pdflatex main.tex
biber main
pdflatex main.tex
pdflatex main.tex
```
Or with `latexmk` (recommended — handles all passes automatically):
```
latexmk -pdf main.tex
```

**Clean build artifacts:**
```
latexmk -C
```

**Regenerate a figure from its Python script:**
```
cd figures_new
python scripts/fig31_pipeline.py      # or any other script in scripts/
```
The scripts write `.png` and `.pdf` outputs directly into `figures_new/`.

**Activate the figures Python environment (Windows):**
```
figures_new\.venv\Scripts\Activate.ps1
```

## Repository Structure

```
main.tex                    # Root document — includes all chapters and packages
references.bib              # BibLaTeX bibliography (IEEE style, sorted by citation order)
frontmatter/                # Cover, declaration, acknowledgements, abstract
chapters/                   # 01–07: Introduction through Conclusion
appendices/                 # Appendix A (implementation), Appendix B (extra results)
figures/                    # Static raster/vector figures
figures_new/                # Figures generated by Python scripts
  scripts/                  # Matplotlib figure-generation scripts (one per figure)
  fig31_pipeline.*          # Compiled TikZ/PDF output for the pipeline diagram
  .venv/                    # Isolated Python env for figure scripts
```

## Architecture

**LaTeX document:** `main.tex` is the single root. It sets up all packages (newtxtext/newtxmath for fonts, biblatex+biber for references, cleveref for cross-refs, TikZ for diagrams) and `\input`s every other `.tex` file. Changes to packages or global formatting belong in `main.tex`.

**Two figure pipelines:**
1. **TikZ** — diagrams authored directly as `.tex` files in `figures_new/` and compiled via `pdflatex` standalone (e.g. `fig31_pipeline.tex`).
2. **Python/Matplotlib** — scripts in `figures_new/scripts/` generate `.png`/`.pdf` files output to `figures_new/`. The venv in `figures_new/.venv/` isolates these dependencies.

Both pipelines output to paths listed in `\graphicspath{{figures/}{figures_new/}{images/}{outputs/figures/}}` so `\includegraphics` can find them without path prefixes.

**Bibliography:** Uses `biblatex` with `backend=biber` and `style=ieee`. Run `biber main` (not `bibtex`) between `pdflatex` passes. All citations go in `references.bib`.

## Key Conventions

- `\code{...}` macro (defined in `main.tex`) renders inline code/variable names in `\texttt`.
- `amssymb` is intentionally **not** loaded — it conflicts with `newtxmath`. Use math symbols from `newtxmath` or `amsmath` only.
- Cross-references use `\cref{}` / `\Cref{}` from `cleveref` (not bare `\ref{}`).
- Chapter labels follow the pattern `\label{ch:slug}`, section labels `\label{sec:slug}`.
- Figure labels follow `\label{fig:slug}`.
